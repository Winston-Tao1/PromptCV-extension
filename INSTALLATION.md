# PromptCV 安装与测试指南 🚀

## 📋 安装步骤

### 第一步：生成图标文件

1. **打开图标生成器**
   - 浏览器中已自动打开 `generate-icons.html` 文件
   - 如果没有打开，请双击该文件手动打开

2. **下载图标**
   - 点击页面上的"📦 下载全部图标"按钮
   - 系统会自动下载4个PNG图标文件：
     * icon16.png
     * icon32.png
     * icon48.png
     * icon128.png

3. **保存图标文件**
   - 将下载的4个图标文件移动到项目的 `icons/` 文件夹中
   - 确保文件名完全一致

### 第二步：加载Chrome扩展

1. **打开Chrome扩展管理页面**
   - 在Chrome浏览器地址栏输入：`chrome://extensions/`
   - 或通过菜单：更多工具 → 扩展程序

2. **启用开发者模式**
   - 点击页面右上角的"开发者模式"开关
   - 确保开关处于开启状态（蓝色）

3. **加载扩展**
   - 点击"加载已解压的扩展程序"按钮
   - 选择项目根目录：`D:\VS code files\PromptCV-extension`
   - 点击"选择文件夹"

4. **验证安装**
   - 扩展应该出现在扩展列表中
   - 图标应该显示在Chrome工具栏右上角
   - 如果没有显示，点击拼图图标查看所有扩展

### 第三步：测试功能

#### 基础功能测试

1. **打开扩展**
   - 点击工具栏中的PromptCV图标
   - 弹窗应该显示，包含彩虹边框动画

2. **添加提示词**
   - 切换到"添加"标签页
   - 选择一个应用（如DeepSeek）
   - 输入测试提示词：
     ```
     请帮我优化这段代码，关注性能和可读性。
     ```
   - 点击"保存并复制"
   - 应该看到成功提示："✅ 提示词已保存并复制"

3. **查看全部提示词**
   - 切换到"全部"标签页
   - 应该看到刚才添加的提示词卡片
   - 卡片应该显示应用徽章、内容预览和时间

4. **测试复制功能**
   - 点击提示词卡片上的"复制"按钮
   - 应该看到提示："✅ 已复制到剪贴板"
   - 在任意文本编辑器中粘贴（Ctrl+V）验证

5. **查看历史记录**
   - 切换到"历史"标签页
   - 应该看到刚才复制的提示词
   - 显示复制时间（如"刚刚"）

6. **测试收藏功能**
   - 在任意提示词卡片上点击右上角的☆图标
   - 图标应该变为⭐（金色）
   - 显示提示："⭐ 已收藏"
   - 切换到"收藏"标签页，应该看到该提示词

#### 高级功能测试

7. **测试标签切换**
   - 在各个标签页之间切换
   - 动画应该流畅，内容正确显示
   - 活动标签应该有蓝色下划线

8. **测试空状态**
   - 如果某个标签页没有内容
   - 应该显示友好的空状态提示
   - 带有相应的图标和说明文字

9. **测试20条限制**
   - 添加超过20条提示词
   - 最旧的提示词应该自动被删除
   - 始终保持20条以内

10. **测试数据持久化**
    - 关闭扩展弹窗
    - 重新打开扩展
    - 之前添加的数据应该还在

#### UI/UX测试

11. **验证设计元素**
    - ✅ 四边有流动的彩虹边框
    - ✅ 白色背景，Google配色
    - ✅ 卡片悬停效果（阴影、上移）
    - ✅ 按钮悬停变色
    - ✅ Toast通知动画流畅

12. **测试响应性**
    - 调整浏览器窗口大小
    - 滚动内容区域
    - 所有元素应该正常显示

## 🐛 故障排除

### 问题1：扩展图标不显示
**解决方案：**
- 确认 `icons/` 文件夹中有所有4个图标文件
- 在扩展管理页面点击刷新按钮
- 如果还不行，尝试移除并重新加载扩展

### 问题2：弹窗无法打开
**解决方案：**
- 右键点击扩展图标 → "检查弹出内容窗口"
- 查看Console中是否有错误信息
- 确认所有文件（HTML、CSS、JS）都存在

### 问题3：样式显示异常
**解决方案：**
- 确认 `styles.css` 文件已正确链接
- 检查浏览器Console是否有CSS加载错误
- 清除浏览器缓存后重试

### 问题4：数据无法保存
**解决方案：**
- 检查 `manifest.json` 中是否有 `storage` 权限
- 查看Console是否有存储相关错误
- 确认Chrome已登录账户（用于同步）

### 问题5：复制功能不工作
**解决方案：**
- 确保浏览器允许剪贴板访问
- 检查是否在HTTPS环境（本地开发可能需要特殊处理）
- 查看Console错误信息
